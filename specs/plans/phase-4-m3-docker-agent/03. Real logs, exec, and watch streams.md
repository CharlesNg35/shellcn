# Phase 4 · Step 3 — Real logs, exec, and watch streams

**Milestone:** M3 · **Status:** ☐ Not started
**Depends on:** 4.2, 3.3 · **Spec refs:** [v2 §7.3](../../v2.md), [v2 §8.1](../../v2.md), [v2 §12](../../v2.md)

## Goal
Validate the streaming + live-update panels for Docker.

## Checklist
- [ ] Container logs WS (follow, `since`, `tail`) → `LogStreamPanel` (real)
- [ ] Container exec WS (TTY, resize) → `TerminalPanel` (reuses 3.3)
- [ ] Docker events → live table/tree patching via `ResourceEvent`
- [ ] Backpressure + reconnect on all streams

## Definition of done
Live logs stream and follow; an exec terminal into a container works; the container table updates in place when a container changes state.

## Notes
This proves `LogStreamPanel` and the watch path against real data — the second streaming validation after SSH.
