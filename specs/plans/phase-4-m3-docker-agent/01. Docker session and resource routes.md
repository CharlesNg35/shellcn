# Phase 4 · Step 1 — Docker session and resource routes

**Milestone:** M3 · **Status:** ☐ Not started
**Depends on:** Phase 2 · **Spec refs:** [v2 §8.2](../../v2.md), [v2 §7.2](../../v2.md), [v2 §7.3](../../v2.md)

## Goal
Docker session via `cfg.Net.DialContext` (so direct and agent share one path) plus
the resource list/inspect routes.

## Checklist
- [ ] `dockerclient.NewClientWithOpts(WithDialContext(cfg.Net.DialContext))`
- [ ] List routes (paginated): containers, images, volumes, networks
- [ ] Inspect routes (per resource)
- [ ] Watch route via Docker events → `ResourceEvent`
- [ ] `HealthCheck` (ping daemon) + `Close`

## Definition of done
List and inspect Docker resources from a test daemon over **direct** transport.

## Notes
Treat Docker connections as `privileged` by default — socket access is root-equivalent (§9.6).
