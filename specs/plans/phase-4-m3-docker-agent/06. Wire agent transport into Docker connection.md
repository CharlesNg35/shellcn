# Phase 4 · Step 6 — Wire agent transport into Docker connection

**Milestone:** M3 · **Status:** ☐ Not started
**Depends on:** 4.1, 4.5 · **Spec refs:** [v2 §8.2](../../v2.md), [v2 §8.3](../../v2.md)

## Goal
Implement the **agent** `NetTransport` dialer (routes through the tunnel) and prove
the Docker plugin works unchanged in agent mode.

## Checklist
- [ ] Agent-mode `cfg.Net.DialContext` opens a stream over the connection's tunnel
- [ ] No change to Docker plugin code vs. direct mode (same `Connect`)
- [ ] Failover/reconnect when the tunnel drops; surface status to UI
- [ ] Idle handling consistent with direct sessions

## Definition of done
A Docker connection in **agent** mode lists containers, streams logs, and execs — using the **same** plugin code path as direct mode, with zero plugin changes.

## Phase 4 exit (M3)
Sidebar-tree + resource detail + watch streams + reverse connectivity (L4 agent) + privileged-risk handling all proven. Update [`../README.md`](../README.md) Phase 4 to ✅.
