## Goal
Wire the existing **agent** `NetTransport` dialer into Docker and prove the
Docker plugin works unchanged in agent mode.

## Checklist
- [x] Agent-mode `cfg.Net.DialContext` opens a stream over the connection's tunnel for Docker API calls
- [x] No change to Docker plugin code vs. direct mode (same `Connect`)
- [x] Failover/reconnect when the tunnel drops; surface status to UI
- [x] Idle handling consistent with direct sessions

## Definition of done
A Docker connection in **agent** mode lists containers, streams logs, and execs — using the **same** plugin code path as direct mode, with zero plugin changes.
