# Phase 4 · Step 2 — Docker manifest (tree, resources, actions)

**Milestone:** M3 · **Status:** ☐ Not started
**Depends on:** 4.1 · **Spec refs:** [v2 §5](../../v2.md), [v2 §6.3](../../v2.md), [v2 §13](../../v2.md)

## Goal
The sidebar-tree manifest with container detail tabs — proving resource-level
capabilities (no nonsensical connection-level Terminal tab).

## Checklist
- [ ] `LayoutSidebarTree`; tree groups: Containers, Images, Volumes, Networks
- [ ] `container` ResourceType: columns, `Watch`, `ActionIDs`, detail tabs (Overview/Terminal/Logs/Inspect/Env)
- [ ] Actions: start/stop/restart (`write`), remove (`destructive`), exec (`privileged`)
- [ ] `SupportedTransports = [direct, agent]`, `AgentProfile` (unix `/var/run/docker.sock`, privileged)
- [ ] Passes manifest validation (2.2)

## Definition of done
A Docker connection renders the tree + container detail tabs from the projection, replacing the Docker fixture.

## Notes
Terminal/Logs appear **inside** container detail, never at connection level (§3 distinction).
