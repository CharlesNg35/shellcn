# Phase 4 · Step 4 — shellcn-agent binary (L4 tcp/unix)

**Milestone:** M3 · **Status:** ☐ Not started
**Depends on:** Phase 2 · **Spec refs:** [v2 §8.3](../../v2.md)

## Goal
`cmd/agent`: the plugin-agnostic reverse-tunnel proxy. v1 = L4 forwarding only
(`tcp`, `unix`).

## Checklist
- [ ] Outbound, mutually-authenticated TLS connection back to the gateway
- [ ] Multiplexed tunnel (yamux/HTTP2) — many channels over one connection
- [ ] `AgentMode` `tcp` (forward a TCP addr) and `unix` (forward a socket)
- [ ] Forward exactly the declared `ProxyTarget`; refuse anything else (least privilege)
- [ ] Reconnect with backoff; heartbeat
- [ ] Build the `shellcn-proxy` container image
- [ ] Static, pure-Go build (cross-platform)

## Definition of done
The agent connects back to the gateway and forwards a unix socket (`/var/run/docker.sock`) and a TCP address on demand.

## Notes
Generic forwarder only — no per-plugin logic. L7 (`k8s_reverse_proxy`) is added in Phase 7.
