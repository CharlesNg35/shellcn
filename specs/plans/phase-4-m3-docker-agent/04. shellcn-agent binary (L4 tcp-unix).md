# Phase 4 · Step 4 — Harden shellcn-agent L4 tcp/unix against Docker

**Milestone:** M3 · **Status:** ✅ Done (2026-05-26)
**Depends on:** Phase 2 · **Spec refs:** [v2 §8.3](../../v2.md)

## Goal
Harden the existing plugin-agnostic `cmd/agent` reverse-tunnel proxy against the
first real L4 target: Docker's unix socket. The generic tunnel exists from Phase
2; this step proves production behavior and packaging.

## Checklist
- [x] TLS hardening for the outbound gateway connection; document certificate/trust expectations
- [x] Multiplexed tunnel behavior under concurrent Docker API calls
- [x] `AgentMode` `tcp` and `unix` verified against real endpoints
- [x] Forward exactly the declared Docker `ProxyTarget`; refuse anything else (least privilege)
- [x] Reconnect with backoff; heartbeat/keepalive behavior validated under dropped links
- [x] Build the `ghcr.io/charlesng35/shellcn-agent` container image
- [x] Static, pure-Go build (cross-platform)

## Definition of done
The agent reliably connects back to the gateway and forwards Docker's unix socket
(`/var/run/docker.sock`) and a TCP target on demand under concurrent use and reconnects.

## Notes
Generic forwarder only — no per-plugin logic. L7 (`http_proxy`) is added in Phase 7.
