# Phase 4 · Step 4 — Harden shellcn-agent L4 tcp/unix against Docker

**Milestone:** M3 · **Status:** ☐ Not started
**Depends on:** Phase 2 · **Spec refs:** [v2 §8.3](../../v2.md)

## Goal
Harden the existing plugin-agnostic `cmd/agent` reverse-tunnel proxy against the
first real L4 target: Docker's unix socket. The generic tunnel exists from Phase
2; this step proves production behavior and packaging.

## Checklist
- [ ] TLS hardening for the outbound gateway connection; document certificate/trust expectations
- [ ] Multiplexed tunnel behavior under concurrent Docker API calls
- [ ] `AgentMode` `tcp` and `unix` verified against real endpoints
- [ ] Forward exactly the declared Docker `ProxyTarget`; refuse anything else (least privilege)
- [ ] Reconnect with backoff; heartbeat/keepalive behavior validated under dropped links
- [ ] Build the `shellcn-proxy` container image
- [ ] Static, pure-Go build (cross-platform)

## Definition of done
The agent reliably connects back to the gateway and forwards Docker's unix socket
(`/var/run/docker.sock`) and a TCP target on demand under concurrent use and reconnects.

## Notes
Generic forwarder only — no per-plugin logic. L7 (`k8s_reverse_proxy`) is added in Phase 7.
