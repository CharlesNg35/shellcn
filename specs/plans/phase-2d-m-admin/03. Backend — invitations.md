# Phase 2d · Step 3 — Backend invitations

**Milestone:** M-Admin · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2d.1, 2d.2 · **Spec refs:** [v2 §9.1](../../v2.md), [v2 §12.2](../../v2.md)

## Goal
Let admins create account invitations with copyable links and optional
best-effort email delivery.

## Checklist
- [x] Add admin create/list/revoke invitation endpoints
- [x] Add public lookup/accept endpoints for invitation tokens
- [x] Store only token hashes
- [x] Make invitation acceptance single-use with an atomic consume transition
- [x] Report actual email delivery status separately from link creation
- [x] Keep SMTP configured through typed bootstrap config

## Definition of done
An admin can create an invitation, share the generated link, optionally email it
when SMTP is configured, revoke pending invitations, and the public accept path
consumes each valid token at most once.

## Tests (gate)
- [x] Invitation flow test covers create/list/lookup/accept/reuse rejection
- [x] Invitation service tests cover email status and atomic consume behavior
