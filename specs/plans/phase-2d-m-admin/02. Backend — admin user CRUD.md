# Phase 2d · Step 2 — Backend admin user CRUD

**Milestone:** M-Admin · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2.5, 2.6, 2.8 · **Spec refs:** [v2 §9.1](../../project.md), [v2 §12.2](../../project.md)

## Goal
Expose admin-only user management APIs with lockout protections and audit rows.

## Checklist
- [x] Add `/api/admin/users` list/create/update/delete endpoints
- [x] Restrict endpoints to admins
- [x] Protect the root admin from deletion, disablement, or demotion
- [x] Allow only the protected root admin to delete, disable, or demote other admins
- [x] Destroy/reject existing sessions when a user becomes disabled
- [x] Audit user create/update/delete outcomes

## Definition of done
Admins can manage ordinary users safely, the protected root admin cannot be
locked out, non-root admins cannot remove peer admins, and disabled accounts no
longer retain active control-plane access.

## Tests (gate)
- [x] Admin authz tests for list/create/update/delete
- [x] Root-admin protection and peer-admin restriction tests
- [x] Disabled-user existing-session rejection test
