# Phase 7 · Step 1 — Kubernetes session and L7 agent mode

**Milestone:** M6 · **Status:** ☐ Not started
**Depends on:** Phase 2, Phase 4 (agent) · **Spec refs:** [v2 §8.2](../../v2.md), [v2 §8.3](../../v2.md)

## Goal
`client-go` over the **L7** transport — proving `cfg.Net.HTTP()` and the agent's
`k8s_reverse_proxy` mode. This is the case a bare dialer can't handle.

## Checklist
- [ ] Build `rest.Config` from kubeconfig (direct) **or** `cfg.Net.HTTP()` (agent)
- [ ] Add `AgentMode = k8s_reverse_proxy` to the agent: L7 proxy to the API server using its in-cluster ServiceAccount
- [ ] Agent install artifact (k8s-manifest, §8.4): `kubectl apply -f "https://host/api/connections/{connectionID}/agent/enrollments/{enrollmentId}/artifacts/k8s-manifest?ticket=…"` — deploys the agent pod + RBAC; token travels inside the manifest, not the path
- [ ] API discovery; dynamic client for CRDs
- [ ] `HealthCheck` + `Close`

## Definition of done
List namespaces both ways: direct (kubeconfig) and via the agent (in-cluster ServiceAccount).

## Notes
This validates the L4/L7 split (§8.2): `DialContext` is insufficient for client-go — use `HTTP()`.
