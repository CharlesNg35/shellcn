# Phase 7 · Step 1 — Kubernetes session and L7 agent mode

**Milestone:** M6 · **Status:** ✅ Done (2026-05-27)
**Depends on:** Phase 2, Phase 4 (agent) · **Spec refs:** [v2 §8.2](../../v2.md), [v2 §8.3](../../v2.md)

> **Landed.** The agent gained a **generic** `http_proxy` (L7) mode — a
> credential-injecting reverse proxy driven by `ProxyTarget.{Address,TokenFile,
> CAFile}`, with **zero Kubernetes vocabulary** in the agent or transport core
> (the k8s API address + ServiceAccount token/CA paths are declared only by the
> plugin's manifest). `transport.agentNet.HTTP()` returns an L7 base URL +
> RoundTripper tunneling over the existing wss yamux tunnel (no inner TLS —
> already encrypted). The k8s session builds `rest.Config` from a kubeconfig
> (direct, exec-credential plugins rejected) or `cfg.Net.HTTP()` (agent), and
> wires typed + dynamic + discovery + cached-RESTMapper + metrics clients. The
> install artifact is a `kubectl apply -f -` manifest (Namespace + SA +
> cluster-admin binding + Secret carrying the enroll token + hardened
> Deployment); the token rides in the manifest body, never a path. Unit tests
> list namespaces over **both** transports against a fake API server.

## Goal
`client-go` over the **L7** transport — proving `cfg.Net.HTTP()` and the agent's
generic `http_proxy` agent mode. This is the case a bare dialer can't handle.

## Checklist
- [x] Build `rest.Config` from kubeconfig (direct) **or** `cfg.Net.HTTP()` (agent)
- [x] Add generic `AgentMode = http_proxy` support to the agent: an L7
      credential-injecting reverse proxy. Kept **plugin-agnostic** — the agent
      injects a bearer token from `ProxyTarget.TokenFile` and verifies with
      `ProxyTarget.CAFile`; the k8s API address + in-cluster SA paths are
      declared by the plugin manifest, not the agent.
- [x] Agent install artifact (k8s-manifest, §8.4): `kubectl apply -f "<url>?ticket=…"`
      fetches the manifest from a **public, single-use signed-ticket** endpoint;
      the enroll token is **minted into the manifest body at fetch time** (the
      record is created with a non-redeemable placeholder hash), so it never
      appears in a path/query. The delivery is a **generic** `InstallArtifact`
      capability (`Delivery: url` + `Content`) any plugin can use — no
      Kubernetes specifics in the core enrollment service or server.
- [x] API discovery; dynamic client for CRDs (discovery + cached RESTMapper +
      `dynamic.DynamicClient` wired; per-kind CRD listing lands in 7.2)
- [x] `HealthCheck` (context-bound `/version`) + `Close` (stops informers)

## Definition of done
List namespaces both ways: direct (kubeconfig) and via the agent (in-cluster ServiceAccount).

## Notes
This validates the L4/L7 split (§8.2): `DialContext` is insufficient for client-go — use `HTTP()`.
