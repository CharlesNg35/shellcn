# Phase 7 · Step 3 — Pod logs, exec, and port-forward

**Milestone:** M6 · **Status:** ☐ Not started
**Depends on:** 7.2, 3.3, 4.3 · **Spec refs:** [v2 §8.1](../../v2.md), [v2 §12](../../v2.md)

## Goal
Pod-level streaming over the L7 transport (these use SPDY/WebSocket upgrades, not
a raw socket).

## Checklist
- [ ] Pod logs WS (container select, follow, previous) → `LogStreamPanel`
- [ ] Pod exec WS (TTY, resize) → `TerminalPanel`
- [ ] Port-forward (channel) → tunnel panel/action
- [ ] All working via **both** direct (kubeconfig) and **agent** (L7) transport
- [ ] Backpressure + reconnect

## Definition of done
Logs, exec, and port-forward all work against a pod via both direct and agent transport.

## Notes
Confirms the L7 transport handles client-go's stream upgrades — the hardest agent case.
