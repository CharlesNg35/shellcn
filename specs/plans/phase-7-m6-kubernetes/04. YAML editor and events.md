# Phase 7 · Step 4 — YAML editor and events

**Milestone:** M6 · **Status:** ☐ Not started
**Depends on:** 7.0 (dock + create-from-content), 7.2 · **Spec refs:** [v2 §12](../../v2.md), [v2 §6.3](../../v2.md)

## Goal
Resource YAML editing and creation (the real `code_editor` use case) and the
events view — the Lens "Create Resource" + edit-in-dock flow (region 6/7), built
from the generic create-from-content + dock primitives in step 0. Manifest-driven
throughout; no k8s-specific UI.

## Checklist
- [ ] **Edit** resource YAML via `code_editor` opened **in the dock**: load current YAML, schema validation, diff view
- [ ] **Create Resource**: a connection-level manifest action opens a templated `code_editor` in the dock (`templatesRouteId` for starter manifests) and applies arbitrary YAML
- [ ] Server-side apply with **dry-run** option; surface validation errors
- [ ] Events table (namespaced + resource-scoped), live-updating via watch
- [ ] Risk: apply/delete tagged appropriately, confirm on destructive

## Definition of done
Edit a Deployment's YAML in the dock and apply it (dry-run + diff); create a new
resource from a template in the dock; the events table streams cluster events —
all declared in the manifest and rendered by the generic dock/editor, no k8s
frontend.
