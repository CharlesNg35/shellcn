# Phase 3 · Step 4 — Wire the real file browser panel

**Milestone:** M2 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 3.2 · **Spec refs:** [v2 §6](../../v2.md), [v2 §13](../../v2.md)

## Goal
Real SFTP routes behind the generic M0 `FileBrowserPanel`. The frontend already
renders list/preview/download/upload/mkdir/rename/delete from manifest-declared
route IDs; this step implements the backend plugin behavior and audit envelope.

## Checklist
- [x] List + navigate directories (paginated for large dirs)
- [x] Download (streamed) and upload (streamed/multipart)
- [x] Rename / delete / mkdir; show permissions, size, mtime
- [x] Inline file view/edit via `CodeEditorPanel` for text files
- [x] Split and grid browser layouts with modal preview, backed by the same manifest-declared routes
- [x] Errors surfaced clearly (perm denied, not found)

## Definition of done
Browse, upload, and download work against the test server through the same SSH session as the terminal (no re-auth).

## Phase 3 exit (M2)
SSH terminal + SFTP work end-to-end through the wrapper (auth, secret decryption, channel lifecycle, audit, manifest rendering). Update [`../README.md`](../README.md) Phase 3 to ✅.
