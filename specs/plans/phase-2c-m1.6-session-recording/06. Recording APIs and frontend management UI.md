# Phase 2c · Step 6 — Recording APIs and frontend management UI

**Milestone:** M1.6 · **Status:** ☐ Not started
**Depends on:** 2b.4, 2b.5, 2c.2, 2c.4, 2c.5 · **Spec refs:** [v2 §9.2](../../v2.md), [v2 §9.5](../../v2.md), [v2 §12.2](../../v2.md)

## Goal
Expose recording management and playback in the platform UI. Admins can see all
recordings; normal users see recordings for sessions they created, plus any
connection-scoped access allowed by policy.

## Checklist
- [ ] APIs: `GET /api/recordings`, `GET /api/connections/{id}/recordings`, `GET /api/recordings/{id}`, `GET /api/recordings/{id}/content`, `DELETE /api/recordings/{id}`
- [ ] APIs for desktop chunks: create/start, chunk upload, finalize, abort
- [ ] Filters: connection, user, protocol, class, format, status, time range
- [ ] Frontend recording list with role-aware scope: admin all, normal user own
- [ ] Connection activity tab/list shows recordings for that connection when user is authorized
- [ ] Playback views: terminal cast player and WebM video player
- [ ] Stream panels show recording state: not supported, disabled, recording, failed
- [ ] Connection create/edit UI integrates recording policy from 2c.1: off-by-default checkbox/options only when supported
- [ ] Download/delete actions with confirmation and audit

## Definition of done
Users can find and play their own recordings. Admins can search all recordings.
Connection create/edit exposes auto-record only for plugins that declare support,
and stream panels show recording state without protocol-specific UI.

## Tests (gate)
- [ ] API authz tests: admin all, user own, unauthorized denied
- [ ] Frontend component tests: list filters, playback route, recording state badge
- [ ] e2e: create connection with terminal auto-record enabled → open fake terminal stream → recording appears → playback opens
- [ ] Deletion requires authorization and writes audit row

## Notes
Recording UI is platform UI, not plugin UI. Plugins declare capabilities; the core frontend renders the same controls for every protocol.
