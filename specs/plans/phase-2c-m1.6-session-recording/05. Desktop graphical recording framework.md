# Phase 2c · Step 5 — Desktop/graphical recording framework

**Milestone:** M1.6 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2c.1, 2c.2, 2c.3 · **Spec refs:** [v2 §9.5](../../project.md), [v2 §11](../../project.md), [v2 §12](../../project.md)

## Goal
Define a protocol-neutral desktop recording path for VNC/RDP remote desktops,
while being explicit about trust boundaries.

## Checklist
- [x] Desktop recording format: `webm_canvas` (browser capture)
- [x] Browser path: `useDesktopRecorder` captures the panel canvas via `captureStream()` + `MediaRecorder`, uploading ordered chunks
- [x] Chunk APIs: start (`POST /connections/{id}/recordings/desktop`), upload (`/recordings/{id}/chunks`), finalize, abort — server-created id, CSRF/session auth, max-chunk cap
- [x] `webm_canvas` recordings are flagged non-authoritative in metadata
- [x] Playback: WebM via native `<video>` (`VideoPlayer.vue`)
- [x] Browser support/mime fallback; a forced desktop recording on an unsupported browser is denied before connecting

**Out of scope (by design):** plugin-implemented recording. The **core** owns
recording end to end (policy, taps, asciicast, browser WebM, storage, authz,
retention, playback); plugins only **declare** recordable capability in their
manifest. M1.6 is complete without any plugin recording subsystem.

## Definition of done
The platform has a generic desktop recording contract. A noVNC canvas can be recorded to WebM when enabled. Plugins declare capability only; they never implement recording.

## Tests (gate)
- [x] Frontend unit test with mocked `captureStream`/`MediaRecorder` uploads chunks and finalizes (`useDesktopRecorder.test.ts`)
- [x] Server tests validate chunk ordering, abort/finalize transitions, and authz (`TestDesktopChunkFlow`); max chunk size capped via `MaxBytesReader`
- [x] Required desktop recording with unsupported browser produces a clear pre-stream error
- [x] Metadata marks `webm_canvas` recordings as non-authoritative

## Notes
Desktop protocols vary heavily. The core owns metadata, authorization, retention,
capture, and playback. Plugin-native protocol capture is intentionally **out of
scope** for M1.6 — see the "Out of scope" note above.
