# Phase 2c · Step 4 — Terminal asciicast recorder and playback

**Milestone:** M1.6 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2c.2, 2c.3 · **Spec refs:** [v2 §9.5](../../project.md), [v2 §12](../../project.md)

## Goal
Implement the generic terminal recorder using asciicast v2 so SSH, Docker exec,
Kubernetes exec, telnet, and serial can share one recording/playback path.

## Checklist
- [x] Write asciicast v2 header with width/height/timestamp/title/env allowlist (TERM/SHELL/LANG only)
- [x] Write an `idle_time_limit` header so compatible players cap long quiet gaps during playback
- [x] Record upstream terminal output as `o` events
- [x] Record resize as `r` events (`{cols}x{rows}`), driven by `Engine.Resize`
- [x] Browser input (`i`) recording is off by default (engine never enables `captureInput`)
- [x] Incremental per-line write; a recording stays valid if the session ends abruptly
- [x] Frontend playback uses `asciinema-player` 3.15.1 (verified via context7 + websearch); `CastPlayer.vue` caps idle gaps for smoother playback
- [x] Download/content type `application/x-asciicast` via `recording.ContentType` (endpoint wired in 2c.6)

## Definition of done
The core can produce a valid `.cast` file from a fake terminal stream, and the frontend can play it back without knowing whether it came from SSH, Docker exec, Kubernetes exec, telnet, or serial.

## Tests (gate)
- [x] Unit: asciicast header + events, env allowlist (`TestAsciicastHeaderAllowlistsEnv`, `TestAsciicastEncodesEventsAndResize`)
- [x] Unit: resize events encoded as `{cols}x{rows}`
- [x] Unit: input events absent unless explicitly enabled (`TestAsciicastOmitsInputUnlessWritten`)
- [x] Frontend component test: playback view mounts a sample cast (`CastPlayer.test.ts`)
- [x] Golden sample recording checked into `testdata/sample.cast` (`TestAsciicastGolden`)

## Notes
Use asciicast as the primary terminal format because it is compact, incremental, and replayable in browser. `xterm.js` serialization is useful for restoring terminal state, not as the storage format for audit recordings.
