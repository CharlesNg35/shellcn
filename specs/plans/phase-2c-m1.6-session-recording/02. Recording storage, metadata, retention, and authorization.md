# Phase 2c · Step 2 — Recording storage, metadata, retention, and authorization

**Milestone:** M1.6 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2.3, 2.6, 2.9, 2c.1 · **Spec refs:** [v2 §9.2](../../v2.md), [v2 §9.5](../../v2.md), [v2 §11.1](../../v2.md)

## Goal
Persist recording metadata in the control-plane store and recording blobs in a
replaceable storage backend. Define who can see recordings.

## Checklist
- [x] Add `Recording` model: id, user, connection, protocol, route/stream id, class, format, status, started/ended, size, checksum, storage key, retention/expires-at, error
- [x] Add `RecordingStore` interface + GORM/in-memory implementations; include it in `store.Store`
- [x] Add blob storage interface (`recording.BlobStore`): local filesystem default (`LocalBlobStore`, traversal-safe), with S3/object storage shape kept behind the interface
- [x] Authorization rules (`RecordingService`): admin sees all (+ per-user filter); normal users see only recordings they created; connection ownership/manage grants do not expose another user's recordings
- [~] Append audit rows for recording start/stop/finalize/read/delete — read/delete auditing lands with the HTTP handlers in 2c.6; start/finalize/failure with the engine in 2c.3
- [x] Retention policy: OFF by default (`config.recordings.retention_days == 0`), optional per-connection days; `Cleanup` deletes expired blobs + marks metadata discarded
- [x] Never store secrets in metadata (metadata carries no params/secret material)

## Definition of done
Recording metadata and blobs persist separately; listing and reading recordings obey role rules (admin all, non-admin own only); expired recordings are cleaned up safely.

## Tests (gate)
- [x] Store round-trip for metadata on SQLite + memory fake (`testRecordings`)
- [x] Blob storage write/read/delete + traversal rejection with temp dir (`blob_test.go`)
- [x] AuthZ tests: admin all, user own only, connection owner/manage denied for other users' recordings (`TestRecordingAuthZScope`)
- [x] Retention cleanup marks/deletes only expired recordings (`TestRecordingRetentionCleanup`)

## Notes
Recordings are larger and more sensitive than audit rows. They need explicit read authorization, retention, and deletion behavior from the start.
