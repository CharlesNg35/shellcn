# Phase 2c · Step 1 — Recording manifest contract and connection policy

**Milestone:** M1.6 · **Status:** ☐ Not started
**Depends on:** 2.1, 2.2, 2b.1, 2b.5 · **Spec refs:** [v2 §5](../../v2.md), [v2 §9.5](../../v2.md), [v2 §12.2](../../v2.md)

## Goal
Let plugins declare recording support generically, and let connection management
store whether recording is enabled. Recording is **off by default** and is never
enabled just because a panel is `terminal` or `remote_desktop`.

## Checklist
- [ ] Add manifest/projection recording declarations: supported classes (`terminal`, `desktop`), formats (`asciicast_v2`, `webm_canvas`, plugin-native), default policy metadata, and whether server-side authoritative recording is available
- [ ] Validate recording declarations at plugin registration: class/format compatibility, referenced stream kinds, and no unsupported auto defaults
- [ ] Extend connection detail/create/update payloads with recording policy: `disabled`, `manual`, `auto`; default `disabled`
- [ ] Connection create/edit UI shows "Auto record" only when the selected plugin projection declares recording support
- [ ] If multiple recording classes are supported, UI lets the user/admin choose per class; no plugin-specific UI
- [ ] Preserve policy on edit; never silently turn recording on during migration or projection refresh

## Definition of done
The projection tells the frontend exactly what recording options are available for a plugin. Connection create/edit can persist an explicit recording policy, and unsupported plugins show no recording checkbox.

## Tests (gate)
- [ ] Manifest validation rejects invalid recording declarations and accepts terminal + desktop examples
- [ ] Projection golden includes recording capability while still hiding server-only implementation details
- [ ] Connection create/update tests prove default is disabled and unsupported recording policy is rejected
- [ ] Frontend component test: plugin with recording support shows the checkbox; plugin without support does not

## Notes
Plugin declaration is the source of truth. The core may provide generic recorders, but it records only when the plugin says the stream is recordable and the connection/user policy enables it.
