# Phase 2c · Step 1 — Recording manifest contract and connection policy

**Milestone:** M1.6 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2.1, 2.2, 2b.1, 2b.5 · **Spec refs:** [v2 §5](../../v2.md), [v2 §9.5](../../v2.md), [v2 §12.2](../../v2.md)

## Goal
Let plugins declare recording support generically, and let connection management
store whether recording is enabled. Recording is **off by default** and is never
enabled just because a panel is `terminal` or `remote_desktop`.

## Checklist
- [x] Add manifest/projection recording declarations: supported classes (`terminal`, `desktop`), formats (`asciicast_v2`, `webm_canvas`), and whether the core recording path is authoritative. Default-policy metadata intentionally omitted — recording stays off by default; a plugin manifest cannot force it on.
- [x] Validate recording declarations at plugin registration: class/format compatibility, referenced stream kinds, duplicate classes
- [x] Extend connection detail/create/update payloads with recording policy: `disabled`, `manual`, `auto`; default `disabled`
- [x] Connection create/edit UI shows recording options only when the selected plugin projection declares recording support
- [x] If multiple recording classes are supported, UI lets the user/admin choose per class; no plugin-specific UI
- [x] Preserve policy on edit (nil submitted map keeps stored policy); never silently turn recording on during migration or projection refresh

## Definition of done
The projection tells the frontend exactly what recording options are available for a plugin. Connection create/edit can persist an explicit recording policy, and unsupported plugins show no recording checkbox.

## Tests (gate)
- [x] Manifest validation rejects invalid recording declarations and accepts terminal + desktop examples (`TestValidateRecordingAccepts`/`Rejects`)
- [x] Projection golden includes recording capability while hiding server-only `streamIds` (`TestProjectionGolden`, `TestProjectionMatchesContract`)
- [x] Connection create/update tests prove default disabled, policy preserved on edit, and unsupported policy rejected (`TestConnectionRecordingPolicy`)
- [x] Frontend component test: plugin with recording support shows options; without support does not (`ConnectionFormDialog.test.ts`)

## Notes
Plugin declaration is the source of truth. The core may provide generic recorders, but it records only when the plugin says the stream is recordable and the connection/user policy enables it.
