# Phase 2b · Step 4 — Frontend auth gate + global error/authz UX

**Milestone:** M1.5 · **Status:** ✅ Done
**Depends on:** 2.5 (auth API), 2.8 · **Spec refs:** [v2 §9.1](../../v2.md), [v2 §12.2](../../v2.md)

## Goal
Gate the SPA behind login, wire CSRF on mutations, and handle auth/error states
consistently — the app currently assumes an open API.

## Checklist
- [x] `LoginView` (PrimeVue inputs) + `auth` Pinia store; `POST /api/auth/login`
- [x] Bootstrap from `GET /api/auth/me` on load; unauthenticated → redirect to `/login`; router guard on protected routes
- [x] Store the CSRF token from login/me; attach it (`X-CSRF-Token`) to every state-changing request in the API client
- [x] Logout (`POST /api/auth/logout`) clears session + returns to login
- [x] Global API error handling: `401` → re-login; `403` → forbidden toast; CSRF / validation / `agent unavailable` → actionable toast (reuse `useToast`)
- [x] Show the current user + logout in `AppShell`

## Definition of done
Visiting any route while unauthenticated lands on login; after login the app works; mutations carry CSRF; an expired session or forbidden action surfaces clearly and routes back to login when appropriate.

## Tests (gate)
- [x] Component/e2e: unauthenticated redirect, successful login → home, logout → login
- [x] CSRF header present on a mutation; 401 mid-session → redirect; 403 → toast (no crash)
