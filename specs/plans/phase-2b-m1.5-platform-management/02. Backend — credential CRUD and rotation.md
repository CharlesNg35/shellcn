# Phase 2b · Step 2 — Backend credential CRUD + rotation

**Milestone:** M1.5 · **Status:** ✅ Done
**Depends on:** 2.3, 2.4 · **Spec refs:** [v2 §9.3](../../v2.md), [v2 §12.2](../../v2.md)

## Goal
Manage reusable credentials over the API. `CredentialService` already encrypts +
resolves; this exposes create/update(rotate)/delete and the usable-list (the
list endpoint already exists for the `credential_ref` selector).

## Checklist
- [x] `POST /api/credentials` — create (kind, name, identity/principal?, secret material); protocol compatibility is derived from the credential kind catalog, not user input
- [x] `PUT /api/credentials/{id}` — update metadata and/or **rotate** the secret (write-only; blank secret keeps the current one)
- [x] `DELETE /api/credentials/{id}` — block deletion while any connection still references the credential
- [x] Secret material is write-only: never returned; responses use `CredentialSummary` (no encrypted blob, no value)
- [x] Credential kind catalog validates known kinds; broad shared kinds live in core, protocol-owned kinds come from plugin manifests, and protocol compatibility is derived from registered `credential_ref` selectors
- [x] Credential summaries carry derived compatible protocols for display/filtering; manual protocol scopes from create/update requests are ignored
- [x] AuthZ: creator becomes owner; update/delete require owner/admin; audited
- [x] Rotation updates once; every referencing connection resolves the new value on next connect (no per-connection edit)

## Definition of done
A user can create, rotate, and delete a reusable credential; the secret is encrypted at rest and never leaves the API; rotation transparently affects all referencing connections.

## Tests (gate)
- [x] Create→rotate→resolve shows the new value at connect-time; summary never carries secret material
- [x] Delete while referenced → 409 conflict; unauthorized rotate/delete → 403

## Notes
Resolution + the use-grant check already live in `CredentialService` (2.4); this step is the write API around it.
