# Phase 2b · Step 6 — Frontend credential management + sharing UI

**Milestone:** M1.5 · **Status:** ✅ Done
**Depends on:** 2b.2, 2b.3, 2b.4 · **Spec refs:** [v2 §9.2](../../v2.md), [v2 §9.3](../../v2.md), [v2 §12.2](../../v2.md)

## Goal
A page to manage reusable credentials (create/rotate/delete) and a sharing UI for
connections + credentials.

## Checklist
- [x] Credentials view: list (`CredentialSummary`), create (full kind picker from the registry catalog → fields per kind; secret material write-only), rotate (replace secret), delete
- [x] Inline credential creation from a connection `credential_ref` is scoped by the plugin selector: only compatible kinds are offered, the protocol is pre-scoped, and the new credential is selected after creation
- [x] Secret inputs are write-only: show `set`/`not set`, never a value; rotation = enter a new value
- [x] Sharing: on a connection (owner/manage) and a credential (owner) — add a subject with access (`use`/`manage` for connections, `use` for credentials), list current grants, revoke
- [x] Subject lookup/selection by username via the Phase 2b user lookup endpoint; clear feedback via toasts; confirm destructive actions
- [x] Empty/loading/error states; accessible + keyboard-operable

## Definition of done
A user creates a credential, rotates its secret (referencing connections pick up the change), shares a connection/credential with another user at the chosen access, and revokes — all without any secret value ever being shown.

## Tests (gate)
- [x] Component: create form per kind; rotate shows write-only affordance; share/revoke updates the grant list
- [x] e2e: create credential → selectable in a connection's `credential_ref`; share connection → grantee sees it (where testable)
