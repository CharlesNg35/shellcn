# Phase 5 · Step 3 — Real noVNC remote-desktop panel

**Milestone:** M4 · **Status:** ☐ Not started
**Depends on:** 5.2, 1.7 · **Spec refs:** [v2 §8.1](../../v2.md), [v2 §12](../../v2.md), [v2 §11](../../v2.md)

## Goal
Use the generic `remote_desktop` panel with `RemoteDesktopConfig.Engine = "novnc"`
over a byte-piped VNC/RFB stream — the third streaming validation.

## Checklist
- [ ] Plugin obtains a VNC ticket/port from the Proxmox API
- [ ] WS `StreamHandler` byte-pipes browser ↔ VNC server (plugin is just a pipe)
- [ ] Manifest console tab declares `remote_desktop` config `{ "engine": "novnc" }`
- [ ] noVNC RFB client mounts on the panel canvas through the engine branch
- [ ] Scaling/resize, clipboard sync, keyboard/mouse
- [ ] Stream held in `sessions` store; survives tab switch

## Definition of done
A VM console renders and is interactive (keyboard/mouse/clipboard) via noVNC.

## Notes
RDP uses the same panel type with `engine: "guacamole"` once the sidecar-backed
RDP plugin lands. SPICE stays out of scope until there is a maintained browser
engine.
