# Phase 5 · Step 3 — Real noVNC remote-desktop panel

**Milestone:** M4 · **Status:** ☐ Not started
**Depends on:** 5.2, 1.7 · **Spec refs:** [v2 §8.1](../../v2.md), [v2 §12](../../v2.md), [v2 §11 (VNC only)](../../v2.md)

## Goal
Replace the stub `RemoteDesktopPanel` with a working noVNC client over a byte-piped
VNC stream — the third streaming validation.

## Checklist
- [ ] Plugin obtains a VNC ticket/port from the Proxmox API
- [ ] WS `StreamHandler` byte-pipes browser ↔ VNC server (plugin is just a pipe)
- [ ] noVNC RFB client mounts on the panel canvas
- [ ] Scaling/resize, clipboard sync, keyboard/mouse
- [ ] Stream held in `sessions` store; survives tab switch

## Definition of done
A VM console renders and is interactive (keyboard/mouse/clipboard) via noVNC.

## Notes
VNC only in v1; RDP (guacd sidecar) and SPICE are out of scope (§2, §11 caveats).
