# Phase 5 · Step 3 — Real remote-desktop panel with noVNC engine

**Milestone:** M4 · **Status:** ✅ Done (2026-05-26)
**Depends on:** 5.2, 1.7 · **Spec refs:** [v2 §8.1](../../v2.md), [v2 §12](../../v2.md), [v2 §11](../../v2.md)

## Goal
Use the generic `remote_desktop` panel over a byte-piped VNC/RFB stream — the
third streaming validation.

## Checklist
- [ ] Plugin obtains a VNC ticket/port from the Proxmox API
- [ ] WS `StreamHandler` byte-pipes browser ↔ VNC server (plugin is just a pipe)
- [ ] Manifest console tab declares `remote_desktop` with VNC/RFB stream source
- [ ] noVNC RFB client mounts on the panel canvas
- [ ] Scaling/resize, clipboard sync, keyboard/mouse
- [ ] Stream held in `sessions` store; survives tab switch

## Definition of done
A VM console renders and is interactive (keyboard/mouse/clipboard) via noVNC.

## Notes
RDP uses the same panel type by bridging server-side decoded frames to a
synthetic RFB stream. SPICE stays out of scope until there is a maintained
browser renderer.
