# Phase 1 · Step 7 — Stub streaming panels

**Milestone:** M0 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 1.4 · **Spec refs:** [v2 §12](../../project.md), [v2 §15 (M0)](../../project.md)

## Goal

Build the chrome of the streaming panels so layouts render fully — but mark them
**stubbed**. They are validated for real with their first plugin (Phases 3–7),
because a mock WebSocket can't prove xterm/noVNC/backpressure.

## Checklist

- [x] `TerminalPanel.vue` — xterm mount, connects to mock WS (echo)
- [x] `LogStreamPanel.vue` — virtualized log viewer on mock frames
- [x] `RemoteDesktopPanel.vue` — generic remote desktop placeholder, with noVNC loaded lazily for RFB streams
- [x] `QueryEditorPanel.vue` — CodeMirror SQL editor + empty results grid
- [x] `CodeEditorPanel.vue` — CodeMirror (YAML/JSON), renders fixture text
- [x] All heavy libs (xterm/noVNC/CodeMirror) **dynamically imported** — loaded only when the panel mounts (v2 §12.1)
- [x] Visible "stub — validated in later plugin milestones" banner on each

## Definition of done

Every layout renders fully with no console errors; streaming panels are clearly marked as stubs (no false "it works" impression); their libs are absent from the initial bundle.

## Tests (gate)

- [x] Component test: each panel mounts and tears down without leaking timers/sockets
- [x] Build check: xterm/noVNC/CodeMirror load only on panel mount (lazy chunk fetched)
- [x] Note: real streaming behavior is **not** asserted here — that is the e2e gate at M2–M6 (see [`../TESTING.md`](../TESTING.md))

## Phase 1 exit (M0)

All four fixtures render end-to-end from projection JSON with zero per-plugin frontend code, frontend unit tests green. Update [`../README.md`](../README.md) Phase 1 to ✅.
