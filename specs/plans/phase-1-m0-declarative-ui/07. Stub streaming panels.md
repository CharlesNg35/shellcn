# Phase 1 · Step 7 — Stub streaming panels

**Milestone:** M0 · **Status:** ☐ Not started
**Depends on:** 1.4 · **Spec refs:** [v2 §12](../../v2.md), [v2 §15 (M0)](../../v2.md)

## Goal

Build the chrome of the streaming panels so layouts render fully — but mark them
**stubbed**. They are validated for real with their first plugin (Phases 3–7),
because a mock WebSocket can't prove xterm/noVNC/backpressure.

## Checklist

- [ ] `TerminalPanel.vue` — xterm mount, connects to mock WS (echo)
- [ ] `LogStreamPanel.vue` — virtualized log viewer on mock frames
- [ ] `RemoteDesktopPanel.vue` — noVNC canvas placeholder
- [ ] `QueryEditorPanel.vue` — Monaco SQL editor + empty results grid
- [ ] `CodeEditorPanel.vue` — Monaco (YAML/JSON), renders fixture text
- [ ] All heavy libs (xterm/noVNC/Monaco) **dynamically imported** — loaded only when the panel mounts (v2 §12.1)
- [ ] Visible "stub — validated in later plugin milestones" banner on each

## Definition of done

Every layout renders fully with no console errors; streaming panels are clearly marked as stubs (no false "it works" impression); their libs are absent from the initial bundle.

## Tests (gate)

- [ ] Component test: each panel mounts and tears down without leaking timers/sockets
- [ ] Build check: xterm/noVNC/Monaco load only on panel mount (lazy chunk fetched)
- [ ] Note: real streaming behavior is **not** asserted here — that is the e2e gate at M2–M6 (see [`../TESTING.md`](../TESTING.md))

## Phase 1 exit (M0)

All four fixtures render end-to-end from projection JSON with zero per-plugin frontend code, frontend unit tests green. Update [`../README.md`](../README.md) Phase 1 to ✅.
