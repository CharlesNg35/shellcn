# Phase 1 · Step 2 — Author fixture manifests and mock dev server

**Milestone:** M0 · **Status:** ☐ Not started
**Depends on:** 1.1 · **Spec refs:** [v2 §5.2](../../v2.md), [v2 §7.2](../../v2.md), [v2 §7.3](../../v2.md), [v2 §12](../../v2.md)

## Goal

Realistic fixtures + a mock server so the whole frontend is built and demoed
without any real backend.

## Checklist

- [ ] `web/fixtures/{ssh,docker,proxmox,postgres}.json` — full projections (representative shapes)
- [ ] Mock data for each `DataSource`: paginated `Page<Row>`, lazy tree children
- [ ] Stub WS endpoints (echo/canned frames) for streaming panels
- [ ] Mock server as Vite dev middleware: `GET /api/plugins`, `GET /api/plugins/{name}`, data routes, `ws`
- [ ] Env toggle `VITE_API=mock|real` to switch mock ↔ real API
- [ ] Fixtures type-check against `projection.ts` (1.1)

## Definition of done

`GET /api/plugins` and each `/api/plugins/{name}` return valid projections; the four fixtures cover tabs, tree, resource-detail, enroll, and query shapes.

## Notes

Fixtures prove the **declarative** surface only — streaming is faked here and validated for real in later plugin milestones.
