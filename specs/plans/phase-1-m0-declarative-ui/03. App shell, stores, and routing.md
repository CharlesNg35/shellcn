# Phase 1 · Step 3 — App shell, stores, and routing

**Milestone:** M0 · **Status:** ☐ Not started
**Depends on:** 1.1, 1.2 · **Spec refs:** [v2 §12](../../v2.md)

## Goal
The application frame: connection sidebar + single-pane workspace, router, and the
Pinia stores — including the session/channel store that owns live streams.

## Checklist
- [ ] `AppShell.vue`: connection sidebar (active/recent) + workspace area
- [ ] Vue Router routes: home, connection workspace, settings
- [ ] Pinia stores: `connections`, `workspace` (active pane/tab), `sessions` (live streams/channels)
- [ ] **Streams live in the `sessions` store, never inside components** (so tab/pane switches don't drop them)
- [ ] Connection list sourced from `/api/plugins` + fixtures
- [ ] (Split panes deferred — single active pane is enough for M0)

## Definition of done
Navigating to a connection renders the shell; active pane/tab state is held in the store and survives component remounts.

## Notes
Split-pane workspace (recursive layout from the transcript) is a post-M0 enhancement.
