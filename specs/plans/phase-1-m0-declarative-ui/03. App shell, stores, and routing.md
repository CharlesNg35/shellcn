# Phase 1 · Step 3 — App shell, stores, and routing

**Milestone:** M0 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 1.1, 1.2 · **Spec refs:** [v2 §12](../../project.md)

## Goal
The application frame: connection sidebar + single-pane workspace, router, and the
Pinia stores — including the session/channel store that owns live streams.

## Checklist
- [x] `AppShell.vue`: connection sidebar (active/recent) + workspace area
- [x] Vue Router routes: home, connection workspace, settings
- [x] Pinia stores: `connections`, `workspace` (active pane/tab), `sessions` (live streams/channels)
- [x] **Streams live in the `sessions` store, never inside components** (so tab/pane switches don't drop them)
- [x] Connection list sourced from `/api/plugins` + fixtures
- [x] (Split panes deferred — single active pane is enough for M0)

## Definition of done
Navigating to a connection renders the shell; active pane/tab state is held in the store and survives component remounts.

## Notes
Split-pane workspace (recursive layout from the transcript) is a post-M0 enhancement.
