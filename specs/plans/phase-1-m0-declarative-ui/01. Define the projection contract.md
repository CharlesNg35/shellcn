# Phase 1 · Step 1 — Define the projection contract (TypeScript)

**Milestone:** M0 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 0.2 · **Spec refs:** [v2 §5.2](../../v2.md), [v2 §6](../../v2.md), [v2 §7](../../v2.md)

## Goal

The single FE/BE contract: `web/src/types/projection.ts`, mirroring the browser
projection (§5.2) and UI types (§6). Everything the renderer consumes is here.

## Checklist

- [x] `PluginProjection` (identity, layout, tabs, tree, resources, actions, streams)
- [x] `Icon` (`{ type: "name"|"url"|"base64"|"emoji"|"svg", value }`) — used by plugin/tab/tree/action/resource icon fields (§5.1)
- [x] `Tab`, `TreeGroup`, `DataSource` (RouteID + params)
- [x] `FileBrowserConfig` (read/download/upload/mkdir/rename/delete route IDs, writable gating, upload limits)
- [x] `FormPanelConfig` (schema submit route, method, label, resource-interpolated params)
- [x] `CodeEditorConfig` / `QueryEditorConfig` (save/cancel route IDs and editor defaults)
- [x] `RemoteDesktopPanelConfig` (required engine: `novnc` or `guacamole`, resize/clipboard/audio options)
- [x] `ResourceType`, `Column`, `Action` (params + risk + requiresConfirm), `Stream` (kind)
- [x] `PanelType` enum (terminal, file_browser, table, metrics, log_stream, code_editor, document, query_editor, remote_desktop, form, enroll, graph, trace, kv, http_client) — still open to future panel strings (§6.2)
- [x] `Schema`, `Field`, `Condition`/`Rule`, `Validator`, `Option`, `CredentialSelector` (`credential_ref` fields)
- [x] `Page<T>`, `PageRequest`, `ResourceEvent`, enroll/status types
- [x] No server-only fields (handlers, raw paths, permission keys) appear here

## Definition of done

Types compile under TS strict and exhaustively cover §6; a sample fixture object type-checks against them, including a `credential_ref` field.

## Tests (gate)

- [x] Type-level: each fixture (1.2) is assigned to `PluginProjection` and compiles
- [x] Unit: `Icon` discriminated-union narrowing; unknown `PanelType` handled
- [x] Lint forbids server-only fields leaking into this file
