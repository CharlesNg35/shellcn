# Phase 1 · Step 1 — Define the projection contract (TypeScript)

**Milestone:** M0 · **Status:** ☐ Not started
**Depends on:** 0.2 · **Spec refs:** [v2 §5.2](../../v2.md), [v2 §6](../../v2.md), [v2 §7](../../v2.md)

## Goal

The single FE/BE contract: `web/src/types/projection.ts`, mirroring the browser
projection (§5.2) and UI types (§6). Everything the renderer consumes is here.

## Checklist

- [ ] `PluginProjection` (identity, layout, tabs, tree, resources, actions, streams)
- [ ] `Icon` (`{ type: "name"|"url"|"base64"|"emoji", value }`) — used by plugin/tab/tree/action/resource icon fields (§5.1)
- [ ] `Tab`, `TreeGroup`, `DataSource` (RouteID + params)
- [ ] `ResourceType`, `Column`, `Action` (risk + requiresConfirm), `Stream` (kind)
- [ ] `PanelType` enum (terminal, file_browser, table, metrics, log_stream, code_editor, document, query_editor, remote_desktop, form, enroll) — **extensible**: `graph`/`trace`/`kv`/`http_client` are added when their plugin lands (§6.2)
- [ ] `Schema`, `Field`, `Condition`/`Rule`, `Validator`, `Option`
- [ ] `Page<T>`, `PageRequest`, `ResourceEvent`, enroll/status types
- [ ] No server-only fields (handlers, raw paths, permission keys) appear here

## Definition of done

Types compile under TS strict and exhaustively cover §6; a sample fixture object type-checks against them.

## Tests (gate)

- [ ] Type-level: each fixture (1.2) is assigned to `PluginProjection` and compiles
- [ ] Unit: `Icon` discriminated-union narrowing; unknown `PanelType` handled
- [ ] Lint forbids server-only fields leaking into this file

## Notes

This file is the contract the Go projection (Phase 2.2) must match — keep them in lockstep; a golden test guards it later. See [`../TESTING.md`](../TESTING.md).
