# Phase 1 · Step 6 — Declarative panels

**Milestone:** M0 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 1.4, 1.5 · **Spec refs:** [v2 §6.1–6.3](../../v2.md), [v2 §5.5](../../v2.md), [v2 §8.4](../../v2.md)

## Goal
Build the panels that **static data fully exercises** — the genuinely
fixture-validated surface.

## Checklist
- [x] `SchemaForm.vue` — renders `Field[]`, structured `Condition` visibility, validators, `multiselect`, `duration`, `file`, and **secret fields write-only** (show "set / not set"). Doubles as the connection-create form.
- [x] `FormPanel.vue` — renders route-backed schemas and submits editable forms through manifest-declared route IDs, switching to multipart when file fields are present.
- [x] `TablePanel.vue` — PrimeVue DataTable: columns from manifest, cursor pagination, filter/sort, refresh, watch updates, row → detail
- [x] `ResourceTree.vue` — lazy `TreeGroup` → children via DataSource, badges
- [x] `FileBrowserPanel.vue` — generic file listing, preview, download, upload, mkdir, rename, delete from manifest-declared route IDs.
- [x] `DocumentPanel.vue` — JSON tree/raw viewer with refresh, copy, and download controls.
- [x] Streaming panels — xterm/logs/metrics/query/editor/desktop lazy-load heavy libraries and expose generic controls; real protocol fidelity lands with the protocol plugins.
- [x] `DetailView.vue` — header (title/status/actions) + tabs
- [x] `ActionBar.vue` — renders `Action[]` with `risk`/`requiresConfirm` styling + confirm dialog + optional input form
- [x] `EnrollPanel.vue` — install command + copy button + "waiting → online" status poll

## Definition of done
All fixture-backed panels render correctly from the four fixtures; the connection-create form renders SSH's config schema with conditional fields. Writable panels prove their route wiring against the mock API without plugin-specific frontend branches.
