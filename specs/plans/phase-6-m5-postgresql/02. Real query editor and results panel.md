# Phase 6 · Step 2 — Real query editor and results panel

**Milestone:** M5 · **Status:** ✅ Done — 2026-05-26
**Depends on:** 6.1, 1.7 · **Spec refs:** [v2 §12](../../project.md), [v2 §13](../../project.md)

## Goal
Replace the stub `QueryEditorPanel` with a real CodeMirror SQL editor + paginated
results — the fourth streaming/large-data validation.

## Checklist
- [x] CodeMirror-backed executable editor with plugin-declared language/labels
- [x] Execute route: streamed result frames over the generic query panel
- [x] Table data browsing reuses the generic paginated table renderer
- [x] Multi-statement handling with per-statement result metadata in the route payload
- [x] Cancel a running query
- [x] Schema-aware completion
- [x] Integration proof against a live PostgreSQL test database

## Definition of done
Run a query, paginate through results, and cancel a long-running query against a test DB.

## Notes
`Table`/`Data` detail tab reuses the generic paginated table renderer for
browse-table-data. `query_editor` is the executable editor/results panel;
plugins that need an editor without an execute affordance use `code_editor`.
