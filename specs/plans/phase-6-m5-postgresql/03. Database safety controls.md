# Phase 6 · Step 3 — Database safety controls

**Milestone:** M5 · **Status:** ☐ Not started
**Depends on:** 6.2 · **Spec refs:** [v2 §9.6](../../v2.md), [v2 §9.5](../../v2.md)

## Goal
The non-negotiable DB guardrails from §9.6.

## Checklist
- [ ] Read-only mode toggle (per connection)
- [ ] Query timeout + row limit (configurable, enforced server-side)
- [ ] Dangerous-statement detection (`UPDATE/DELETE/TRUNCATE/DROP`) → confirm/approval, tagged `destructive`
- [ ] Optional approval workflow hook for production connections
- [ ] **Every query audited** (statement, rows affected, duration, result)
- [ ] Result redaction hooks where configured

## Definition of done
A destructive statement is gated behind confirm/approval in a protected connection; read-only mode blocks writes; all queries are audited.

## Phase 6 exit (M5)
Real query editor + DB safety proven. Update [`../README.md`](../README.md) Phase 6 to ✅.
