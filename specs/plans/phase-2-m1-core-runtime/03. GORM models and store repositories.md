# Phase 2 · Step 3 — GORM models and store repositories

**Milestone:** M1 · **Status:** ✅ Done (SQLite gate)
**Depends on:** 2.1 · **Spec refs:** [v2 §11.1](../../v2.md), [v2 §14.1](../../v2.md)

## Goal
The cross-database control-plane store via `gorm`, hidden behind repository
interfaces so the ORM never leaks.

## Structure decision (committed)
- Core entities live in **`internal/models`** (package `models`) and **are** the
  GORM models — gorm tags sit directly on them. **No** parallel row/DTO structs +
  `toX/fromX` mappers (rejected as needless complexity; keeps DX high).
- `internal/store` is the **only** package that imports the `gorm` package
  (enforced by an arch test). gorm struct *tags* on `models` don't import gorm,
  so the invariant holds.
- Slice/map columns use `gorm:"serializer:json"`. Secret-ish columns
  (`User.PasswordHash`) are `json:"-"` and the store clears/omits them on read.

## Checklist
- [x] Models in `internal/models`: `User`, `Connection` (+ encrypted inline secret columns), `Credential` (+ encrypted secret material), `Grant`, `CredentialGrant`, `AuditEntry`, `Snippet`, `AgentEnrollment`, `Preference` (+ `Role`/`Access`/status enums)
- [x] Repository interfaces in `internal/store`: `UserStore`, `ConnectionStore`, `CredentialStore`, `GrantStore`, `CredentialGrantStore`, `AuditStore`, `SnippetStore`, `PreferenceStore`, `EnrollmentStore`
- [x] Driver wiring: `glebarez/sqlite` (pure-Go default) + `gorm.io/driver/postgres` (pgx) + `gorm.io/driver/mysql` — never the cgo `gorm.io/driver/sqlite`
- [x] In-memory fakes implementing the interfaces (`store.NewMemory()`) for unit tests
- [x] Startup auto-migration via `AutoMigrate`: create missing tables/indexes and apply additive schema changes only
- [x] Guardrail: destructive schema/data changes are never automatic
- [x] Credential references: connections point at reusable credentials by id without duplicating secret material
- [x] Credential grants: `use` access permits connect-time resolution but never readback of secret values
- [x] **Nothing outside `internal/store` imports the `gorm` package** (arch test)

## Definition of done
A fresh **SQLite** database boots and creates its schema automatically via `AutoMigrate`; CRUD works for users, connections, credentials, and grants on SQLite (the M1 gate); the same store suite is parameterized to also run on Postgres + MySQL; no `gorm` import leaks out of `store`.

## Tests (gate)
- [x] Store suite green on **SQLite** + the in-memory fake (per-PR gate — fast), including connection → credential reference CRUD
- [x] Same suite runs on **Postgres + MySQL** when `TEST_POSTGRES_DSN`/`TEST_MYSQL_DSN` are set (nightly / M1 hardening — see [`../TESTING.md`](../TESTING.md))
- [x] Arch test: no package outside `internal/store` imports the `gorm` package

## Notes
Secrets are encrypted in the service layer (2.4) before reaching the store — the store only sees ciphertext and stable credential references. SQLite stays the zero-config default; cross-DB is opt-in (v2 §11.1). Versioned migrations remain a later hardening option once the schema stabilizes.
