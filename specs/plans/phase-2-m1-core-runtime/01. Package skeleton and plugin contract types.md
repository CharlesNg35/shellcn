# Phase 2 · Step 1 — Package skeleton and plugin contract types

**Milestone:** M1 · **Status:** ☐ Not started
**Depends on:** 0.1 · **Spec refs:** [v2 §5](../../v2.md), [v2 §6](../../v2.md), [v2 §8](../../v2.md)

## Goal
Define the Go plugin contract in `internal/plugin`, matching `projection.ts` (1.1)
so FE and BE agree on shape.

## Checklist
- [ ] `Plugin` (Manifest, Routes, Connect), `Session`/`Channel`/`ClientStream`
- [ ] `Manifest` (+ `Capability`, `Transport`, `AgentMode`, `AgentProfile`, `ProxyTarget`, `InstallArtifact`)
- [ ] `Icon` (+ `IconType` name/url/base64/emoji) — used by Manifest/Tab/TreeGroup/Action/Resource icon fields (§5.1)
- [ ] `Route` (+ `Method`, `RiskLevel`), `Handler`, `StreamHandler`
- [ ] `RequestContext` (`Param`, `Query`, `Bind`, `Page`) — typed, no `map[string]any` assertions
- [ ] `ConnectConfig` + `NetTransport` (`DialContext` L4, `HTTP()` L7)
- [ ] `Schema`/`Field`/`Condition`, `Tab`/`TreeGroup`/`DataSource`, `ResourceType`/`Action`/`Stream`/`ResourceRef`
- [ ] `PanelType` constants incl. the extensible set (`graph`/`trace`/`kv`/`http_client`) (§6.2)
- [ ] `registry` (Register/Get/All)

## Definition of done
Types compile; a golden test round-trips a manifest to the exact JSON shape `projection.ts` expects.

## Tests (gate)
- [ ] Golden: `Manifest` (with structured `Icon`) JSON-marshals to the `projection.ts` shape (1.1) — FE/BE drift guard
- [ ] Unit: `RequestContext.Bind` decodes + validates typed structs (e.g. JSON number → int) with **no panic** on missing/wrong type
- [ ] `go vet` + `-race` clean

## Notes
Plugin structs hold **no** per-connection state (§5). Per-connection state lives in `Session`. See [`../TESTING.md`](../TESTING.md).
