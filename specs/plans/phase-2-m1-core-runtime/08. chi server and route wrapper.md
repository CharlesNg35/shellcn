## Goal

Mount plugin routes and wrap every one with the full middleware chain; serve the
embedded frontend.

## Checklist

- [ ] chi router; mount plugin routes at `/api/connections/{id}/x/{routeID}`
- [ ] Wrapper chain: authn ‚Üí authz (permission + risk) ‚Üí session resolve ‚Üí `Bind`/validate input ‚Üí audit ‚Üí handler ‚Üí normalize error/response
- [ ] Audit is called through an `AuditSink` interface; use a noop sink until the real writer lands in 2.9
- [ ] WS upgrade path (ticket + origin) for `Method == WS`, bridging `ClientStream` ‚Üî handler
- [ ] Resolve `RouteID` ‚Üí the route's registered path template; fill `{name}` from **`?p.<name>`** query params; expose via `rc.Param` (¬ß7.1)
- [ ] Serve `web/dist` with SPA fallback; API routes take priority
- [ ] Consistent JSON error envelope

## Definition of done

A request flows through every middleware stage; `p.`-params resolve into the path template; errors are normalized; the embedded UI is served; WS upgrades work behind a param-scoped ticket.

## Tests (gate)

- [ ] `RouteID` + `p.` params resolve correctly; unknown RouteID‚Üí404
