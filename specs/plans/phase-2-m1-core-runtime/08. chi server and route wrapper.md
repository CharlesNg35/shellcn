# Phase 2 · Step 8 — chi server and route wrapper

**Milestone:** M1 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2.2, 2.4, 2.5, 2.6, 2.7 · **Spec refs:** [v2 §4](../../v2.md), [v2 §5.2](../../v2.md), [v2 §5.3](../../v2.md), [v2 §7.1](../../v2.md)

## Goal

Mount plugin routes and wrap every one with the full middleware chain; serve the
embedded frontend.

## Checklist

- [x] chi router; mount plugin routes at `/api/connections/{id}/x/{routeID}`
- [x] Wrapper chain: authn → authz (permission + risk) → session resolve → `Bind`/validate input → audit → handler → normalize error/response
- [x] Audit is called through an `AuditSink` interface; use a noop sink until the real writer lands in 2.9
- [x] WS upgrade path (ticket + origin) for `Method == WS`, bridging `ClientStream` ↔ handler
- [x] Resolve `RouteID` → the route's registered path template; fill `{name}` from **`?p.<name>`** query params; expose via `rc.Param` (§7.1)
- [x] Bind both JSON and declared `multipart/form-data` route inputs; expose uploaded files to handlers without logging file bytes or secret form parts.
- [x] Serve `web/dist` with SPA fallback; API routes take priority
- [x] Consistent JSON error envelope

## Definition of done

A request flows through every middleware stage; `p.`-params resolve into the path template; errors are normalized; the embedded UI is served; WS upgrades work behind a param-scoped ticket.

## Tests (gate)

- [x] Wrapper order test: unauthenticated→401, unauthorized(risk)→403, missing session→503, bad input→400, happy→200 + audit row
- [x] Declared route input schema is validated by the wrapper before the handler runs
- [x] Denied authorization emits an audit row with route id/risk/params
- [x] `RouteID` + `p.` params resolve correctly; unknown RouteID→404
- [x] WS: missing/expired/param-mismatched ticket rejected at upgrade

## Notes

Handlers never touch `http.ResponseWriter` — they return `(any, error)` (§4).
This step defines the audit enforcement point; 2.9 replaces the noop sink with
the append-only audit writer.
