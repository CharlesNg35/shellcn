# Phase 2 · Step 5 — Authentication and sessions

**Milestone:** M1 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2.3 · **Spec refs:** [v2 §9.1](../../v2.md), [v2 §7.4](../../v2.md)

## Goal
Local accounts + platform session + WebSocket ticket auth. OIDC behind an
interface from day one (impl later).

## Checklist
- [x] Local accounts; password hashing (argon2id)
- [x] Platform session cookie: `HttpOnly`, `SameSite`; CSRF token for state-changing HTTP
- [x] `Authenticator` interface with OIDC/OAuth2 stub implementation
- [x] WS tickets: short-lived (~30s), single-use, signed, scoped to **(connectionID, routeID, params, user)** — params bound so a ticket can't be replayed against a different resource (§7.4)
- [x] WS upgrade validates ticket, **that bound params match the request**, + **origin** check
- [x] Optional MFA/TOTP hook (interface only)

## Definition of done
Login works; protected HTTP routes require a valid session + CSRF; WS upgrades require a valid ticket; a ticket minted for one resource is rejected on another; origin enforced.

## Tests (gate)
- [x] Ticket: expiry, single-use (replay rejected), param-mismatch rejected, bad origin rejected
- [x] CSRF required on state-changing HTTP; argon2id verify round-trip

## Notes
Browsers can't set `Authorization` on WS — tickets are mandatory, not optional (§7.4).
