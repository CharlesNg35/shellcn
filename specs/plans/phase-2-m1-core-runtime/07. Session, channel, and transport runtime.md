# Phase 2 · Step 7 — Session, channel, and transport runtime

**Milestone:** M1 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2.1 · **Spec refs:** [v2 §8.1](../../v2.md), [v2 §8.2](../../v2.md)

## Goal

In-memory session/channel registry + the **direct** `NetTransport`. Agent transport
is stubbed here and built in Phase 4.

## Checklist

- [x] Session registry keyed by `(connectionID, ownerScope)`
- [x] Lazy `Connect` on first use; periodic `HealthCheck`; graceful `Close` on shutdown
- [x] Idle timeout; max sessions + channels per user
- [x] Session status snapshot + browser keepalive endpoint for the active workspace
- [x] Short-lived error status for failed connect/health check, cleared by disconnect
- [x] Active browser WS streams pin the session even without an upstream channel
- [x] `OpenChannel` tracking (counters, per-channel cleanup/audit)
- [x] `NetTransport`: direct `DialContext` (L4) + `HTTP()` (L7) implementations
- [x] Tunnel-registry interface (agent dialer wired in 4.6)
- [x] Dead upstream closes channels + signals UI to offer reconnect

## Definition of done

Sessions open/close, browser keepalive preserves the one user/connection session,
failed health is visible as session state, browser streams prevent idle reclaim
while open, channels are tracked, idle timeout reclaims sessions with no channels
or streams, and a direct dial reaches a target.

## Notes

Single-instance, in-memory by design (v2 §2). HA is explicitly out of scope for v1.
