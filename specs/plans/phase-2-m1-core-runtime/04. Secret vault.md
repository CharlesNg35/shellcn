# Phase 2 · Step 4 — Secret vault

**Milestone:** M1 · **Status:** ☐ Not started
**Depends on:** 2.3 · **Spec refs:** [v2 §9.3](../../v2.md)

## Goal
Encrypt connection secrets at rest behind a `SecretStore` interface (OpenBao
drop-in later).

## Checklist
- [ ] `SecretStore` interface (`internal/secrets`)
- [ ] Local AES-256-GCM vault; master key from env/file (KMS-ready)
- [ ] Per-record data key wrapped by the master key
- [ ] Encrypt `Field.Secret == true` values; persist only ciphertext
- [ ] **Write-only API**: never return secret values; expose "set / not set"
- [ ] Redaction helpers used by logs + audit
- [ ] Master-key rotation path documented

## Definition of done
Secrets persist as ciphertext, never leave the API, never appear in logs/audit; encrypt→decrypt round-trip test passes.

## Notes
Encryption happens in the service layer, independent of the DB engine (2.3).
