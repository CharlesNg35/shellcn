# Phase 2 · Step 6 — Authorization with Casbin

**Milestone:** M1 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2.3, 2.5 · **Spec refs:** [v2 §9.2](../../project.md)

## Goal
RBAC + per-connection ownership/grants + action-risk enforcement, embedded with
Casbin (OPA later, additive).

## Checklist
- [x] Casbin model + additive policy storage in the store
- [x] Roles: `admin` / `operator` / `viewer` (extensible)
- [x] Per-connection ownership + explicit sharing grants
- [x] Enforce each route's `Permission` + `Risk` (deny-by-default)
- [x] Enforcement point lives in the route wrapper (2.8), checked per request incl. WS actions
- [x] Tests cover the risk matrix (viewer blocked from destructive/privileged; owner allowed)

## Definition of done
A viewer is denied destructive/privileged routes; an owner is allowed; risk-matrix tests pass.

## Notes
Risk levels come from the route metadata (single source of truth) — authz never trusts client-supplied risk.
