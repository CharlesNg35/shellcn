# Phase 2 · Step 6 — Authorization with Casbin

**Milestone:** M1 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2.3, 2.5 · **Spec refs:** [v2 §9.2](../../v2.md)

## Goal
RBAC + per-connection ownership/grants + action-risk enforcement, embedded with
Casbin (OPA later, additive).

## Checklist
- [ ] Casbin model + policy storage (in the store)
- [ ] Roles: `admin` / `operator` / `viewer` (extensible)
- [ ] Per-connection ownership + explicit sharing grants
- [ ] Enforce each route's `Permission` + `Risk` (deny-by-default)
- [ ] Enforcement point lives in the route wrapper (2.8), checked per request incl. WS actions
- [ ] Tests cover the risk matrix (viewer blocked from destructive/privileged; owner allowed)

## Definition of done
A viewer is denied destructive/privileged routes; an owner is allowed; risk-matrix tests pass.

## Notes
Risk levels come from the route metadata (single source of truth) — authz never trusts client-supplied risk.
