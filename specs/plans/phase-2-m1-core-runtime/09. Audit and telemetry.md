# Phase 2 · Step 9 — Audit and telemetry

**Milestone:** M1 · **Status:** ✅ Done (2026-05-25)
**Depends on:** 2.3, 2.8 · **Spec refs:** [v2 §9.5](../../v2.md), [v2 §10](../../v2.md)

## Goal
Append-only audit log + structured logs/metrics/health so the gateway is
observable from day one.

## Checklist
- [ ] Audit writer keyed by route `AuditEvent`: who, what, when, connection, params (secrets redacted), result, risk
- [ ] Append-only storage (no updates/deletes on audit rows)
- [ ] `log/slog` structured logging; request/correlation IDs
- [ ] Prometheus metrics: sessions/channels open, action latency, WS connections, failed authorizations, secret-access counts, plugin health
- [ ] `/healthz` (incl. per-plugin `HealthCheck`) and `/metrics`

## Definition of done
Every wrapped route emits exactly one audit record; `/metrics` and `/healthz` expose the counters and health.

## Notes
OpenTelemetry traces are additive/optional (§10) — not required for M1.
