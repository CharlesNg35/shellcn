# Phase 0 · Step 1 — Initialize Go module and repo skeleton

**Milestone:** — (bootstrap) · **Status:** ☐ Not started
**Depends on:** none · **Spec refs:** [v2 §14](../../v2.md)

## Goal
Create the Go module and the `internal/` package layout so every later layer has a
home and `go build ./...` is green from day one.

## Checklist
- [ ] `go mod init MODULE` (choose the real module path, e.g. `github.com/charlesng/shellcn`)
- [ ] Create packages: `cmd/server`, `cmd/agent`, `plugins/`, and
      `internal/{domain,store,service,server,plugin,session,transport,auth,policy,secrets,audit,telemetry,config}`
- [ ] `cmd/server/main.go` stub that builds and prints version/flags (`--dev`, `--addr`)
- [ ] Confirm the existing `web/embed.go` (`package web`) still compiles with a placeholder `web/dist`
- [ ] Add `.editorconfig`, `LICENSE`, top-level `README.md`
- [ ] First commit

## Definition of done
`go build ./...` succeeds and `./shellcn --version` runs. Package layout matches v2 §14.

## Notes
Keep `domain` dependency-free (pure types/interfaces). Nothing imports `internal/server` except `cmd/server`.
